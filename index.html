Detailed Theory Primer

In formal language theory, a language can be described either by a generator or by a recognizer. For context-free languages (CFLs), the generator is a Context-Free Grammar (CFG) and the recognizer is a Pushdown Automaton (PDA). This is one of the most important equivalence results in automata theory.

CFG: G = (V, Sigma, R, S)

PDA: M = (Q, Sigma, Gamma, delta, q0, Z0, F)

Equivalence: L(G) = L(M) for every CFL.

What CFGs Do

CFGs build strings by repeatedly replacing variables using production rules. A derivation starts from S and ends when only terminals remain.

  • Great for expressing recursive syntax.
  • Common in parser design and programming language grammars.
  • Naturally represented as parse trees.

What PDAs Do

PDAs read input symbols while using a stack for extra memory. The stack enables matching nested or balanced patterns that finite automata cannot handle.

  • Can push symbols to remember open structure.
  • Can pop symbols to verify matched structure.
  • Accepts by final state in this project.

CFG vs PDA at a Glance

Aspect CFG PDA
Main View Generative (productions) Operational (state + stack moves)
Memory Implicit via recursion Explicit stack (Gamma)
Typical Output Derivation / parse tree Run trace / configurations
Use Cases Syntax specification Language recognition

How This Converter Builds Equivalence

CFG -> PDA Construction

  1. Start with bottom marker on stack.
  2. Push grammar start symbol S.
  3. For each production A -> alpha, add epsilon transition replacing A by alpha.
  4. For each terminal a, read a and pop a.
  5. Accept when input is consumed and bottom marker remains.

PDA -> CFG Construction

  1. Create variables [p,X,q] for states p,q and stack symbol X.
  2. Interpret [p,X,q] as all strings taking PDA from p to q while popping X.
  3. Translate transitions into grammar productions over these variables.
  4. Add start productions from PDA start to accept states.
  5. Simplify by removing duplicate, non-generating, and unreachable rules.

Note: The app demonstrates standard educational constructions. It prioritizes readability and traceability over heavy optimization.

Worked Language Example

Consider L = { a^n b^n | n >= 0 }. A classic CFG is S -> aSb | epsilon. In a PDA view, each a pushes a marker and each b pops one marker. Acceptance occurs when input ends and the stack returns to the bottom marker.

Valid strings: epsilon, ab, aabb, aaabbb

Invalid strings: aab, abb, aaabb

Why invalid: stack depth and remaining input do not match.
